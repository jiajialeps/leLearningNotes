
###01.使用 `CASE WHEN` 替换 `UNION ALL`
```sql
CREATE TEMPORARY TABLE tmp_card_cust_polymeric_0001 AS
SELECT
    cbi_customer_id,
    cct_cust_class
FROM
    (
        SELECT
            cbi_customer_id,
            cbi_customer_class
        FROM
            customer_basic_info
        WHERE
            biz_dt = "20240814"
    ) cbi
    JOIN tmp_card_cust_priority_high_0001 tccphl ON cbi.cbi_customer_id = tccphl.card_customer_id
WHERE
    cbi.cbi_customer_class != tccphl.cct_cust_class
UNION ALL
SELECT
    cbi.cbi_customer_id,
    "" AS cct_cust_class
FROM
    (
        SELECT
            cbi_customer_id
        FROM
            customer_basic_info
        WHERE
            biz_dt = "20240814"
            AND cbi_customer_class IN ('W', 'V', 'Q')
    ) cbi
    LEFT JOIN tmp_card_cust_priority_high_0001 tccph1 ON cbi.cbi_customer_id = tccph1.card_customer_id
WHERE
    tccph1.card_customer_id IS NULL;
```

优化后

```sql
CREATE TEMPORARY TABLE tmp_card_cust_polymeric_0001 AS
SELECT 
    cbi.cbi_customer_id,
    CASE 
        WHEN tccphl.card_customer_id IS NOT NULL AND cbi.cbi_customer_class != tccphl.cct_cust_class 
            THEN tccphl.cct_cust_class
        WHEN tccphl.card_customer_id IS NULL AND cbi.cbi_customer_class IN ('W', 'V', 'Q') 
            THEN ''
        ELSE NULL
    END AS cct_cust_class
FROM customer_basic_info cbi
LEFT JOIN tmp_card_cust_priority_high_0001 tccphl 
    ON cbi.cbi_customer_id = tccphl.card_customer_id
WHERE cbi.biz_dt = "20240814";

```

### 主要优化点：

1. **使用 `CASE WHEN` 替换 `UNION ALL`**
    - 原 SQL 的第一部分是在 `JOIN` 之后筛选 `cbi_customer_class != cct_cust_class`，这个逻辑可以在 `CASE WHEN` 里处理。
    - 原 SQL 的第二部分是筛选 `cbi_customer_class` 在 `('W', 'V', 'Q')` 且 `tccph1.card_customer_id IS NULL`，也可以用 `CASE WHEN` 处理。
2. **减少 `UNION ALL` 带来的重复扫描**
    - `customer_basic_info` 只需要扫描一次，避免 `UNION ALL` 带来的额外查询。
3. **保持 `LEFT JOIN` 逻辑**
    - 这样可以保证即使 `tmp_card_cust_priority_high_0001` 没有匹配的记录，仍然可以给 `cct_cust_class` 赋 `""`。

这样优化后，查询性能会更优，因为数据库只需遍历 `customer_basic_info` 一次，而不需要两次查询再 `UNION ALL`。
